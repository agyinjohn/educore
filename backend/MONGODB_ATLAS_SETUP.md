# MongoDB Atlas Setup Guide for EduCore

## Step 1: Create MongoDB Atlas Account

1. Go to https://www.mongodb.com/cloud/atlas
2. Sign up for a free account
3. Create an organization and project

## Step 2: Create a Cluster

1. Click "Create a Deployment"
2. Select **M0 Sandbox** (free tier)
3. Choose a cloud provider (AWS, Google Cloud, or Azure)
4. Select a region close to you
5. Click "Create Deployment"

## Step 3: Setup Network Access & User

### Network Access:
1. Go to "Network Access" in left sidebar
2. Click "Add IP Address"
3. Choose "Allow Access from Anywhere" (for development)
4. Click "Confirm"

### Database User:
1. Go to "Database Access" in left sidebar
2. Click "Add New Database User"
3. Enter username and password (save these!)
4. Select "Autogenerated Secure Password" or create your own
5. Click "Add User"

## Step 4: Get Connection String

1. Go to "Databases" in left sidebar
2. Click "Connect" on your cluster
3. Select "Drivers"
4. Copy the connection string

The string looks like:
```
mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
```

## Step 5: Update .env File

Update your `.env` file in `/Users/apexcode/Desktop/EduCore/backend/.env`:

```env
MONGO_URI=mongodb+srv://your-username:your-password@cluster0.xxxxx.mongodb.net/educore?retryWrites=true&w=majority
```

**Important**: Replace:
- `your-username` - Your MongoDB Atlas username
- `your-password` - Your MongoDB Atlas password (URL encode special characters)
- `cluster0.xxxxx` - Your actual cluster URL

## Step 6: Create Database Collections

The application will auto-create collections when services start. However, you can pre-create them:

```javascript
// In MongoDB Atlas Console
use educore

// Student Service Database
db.createCollection("students")

// Academic Service Database
db.createCollection("classes")
db.createCollection("timetableslots")
db.createCollection("attendances")
db.createCollection("grades")
db.createCollection("assessments")
db.createCollection("exams")
```

## Step 7: Verify Connection

After updating `.env`, run:

```bash
npm run dev:phase1
```

You should see:
```
✓ student-service listening on port 3002
✓ academic-service listening on port 3003
✓ MongoDB connected successfully
```

## Microservices Database Strategy

Since you're using microservices, you have two options:

### Option A: Single Shared Database (Recommended for Phase 1)
```
One MongoDB database "educore"
├── students (student-service)
├── classes (academic-service)
├── timetableslots (academic-service)
├── attendances (academic-service)
├── grades (academic-service)
├── assessments (academic-service)
└── exams (academic-service)
```

**Advantages:**
- Simple to manage
- Easy to query across services (via API)
- Good for Phase 1

**Connection**: Both services use same `MONGO_URI`

### Option B: Separate Databases per Service (Future - Phase 2+)
```
MongoDB Atlas Cluster
├── educore-student (student-service)
│   └── students
├── educore-academic (academic-service)
│   ├── classes
│   ├── timetableslots
│   ├── attendances
│   ├── grades
│   ├── assessments
│   └── exams
├── educore-finance (finance-service) [Phase 2]
└── educore-communication (communication-service) [Phase 3]
```

**Advantages:**
- True microservices isolation
- Independent scaling
- Better for large deployments

**Implementation**: Create separate `MONGO_URI` env vars per service

## Environment Variables for Microservices

### Option A: Shared Database (Current)
```env
# Single connection string for all services
MONGO_URI=mongodb+srv://user:pass@cluster.mongodb.net/educore?retryWrites=true&w=majority
```

All services in `config/db.ts` will use the same URI.

### Option B: Separate Databases (Future)
```env
# Per-service connection strings
STUDENT_DB_URI=mongodb+srv://user:pass@cluster.mongodb.net/educore-student
ACADEMIC_DB_URI=mongodb+srv://user:pass@cluster.mongodb.net/educore-academic
FINANCE_DB_URI=mongodb+srv://user:pass@cluster.mongodb.net/educore-finance
```

Update `services/*/src/config/db.ts` to use respective URIs.

## Troubleshooting

### Connection Refused
- Check `.env` has correct `MONGO_URI`
- Verify username/password are correct
- Check Network Access allows your IP
- Special characters in password must be URL encoded

### Collection Not Found
- Collections auto-create on first insert
- Or manually create via MongoDB Atlas Console

### Permission Denied
- User role should be "Atlas admin" or appropriate
- Verify user has access to the database

### Slow Connections
- MongoDB Atlas M0 is free tier (slower)
- Upgrade to M2 or higher for better performance
- Use M0 for development only

## Connection String Encoding

If your password contains special characters, URL encode them:

```
! → %21
@ → %40
# → %23
$ → %24
% → %25
& → %26
: → %3A
/ → %2F
```

**Example:**
```
Password: my@pass#123
Encoded: my%40pass%23123

Connection: mongodb+srv://user:my%40pass%23123@cluster.mongodb.net/educore
```
